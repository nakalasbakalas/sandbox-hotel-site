## HREFLANG IMPLEMENTATION

### Current Status

**Existing hreflang tags** in `<head>`:
```html
<link rel="alternate" hreflang="th" href="https://www.sandboxhotel.com/?lang=th">
<link rel="alternate" hreflang="en" href="https://www.sandboxhotel.com/?lang=en">
<link rel="alternate" hreflang="zh-CN" href="https://www.sandboxhotel.com/?lang=zh">
<link rel="alternate" hreflang="x-default" href="https://www.sandboxhotel.com/">
```

**Analysis**:
- ✅ Thai: `hreflang="th"` ✓
- ✅ English: `hreflang="en"` ✓
- ⚠️ Chinese: Uses `hreflang="zh"` but URL param is `?lang=zh`
  - Should be `hreflang="zh-CN"` for Simplified Chinese
  - Current implementation may work but is not perfectly aligned

**Recommendation**:
Update line to:
```html
<link rel="alternate" hreflang="zh-CN" href="https://www.sandboxhotel.com/?lang=zh-CN">
```
And update URL parameter handling to support `?lang=zh-CN` (or keep `?lang=zh` and use `hreflang="zh"` consistently)

### x-default Tag

**Current**: Points to root URL (no lang parameter)
**Behavior**: Defaults to Thai (per `I18N_DEFAULT = "th"` in code)

**Best Practice**: `x-default` should point to a language selector page OR the primary market language

**Recommendation**: Current implementation is acceptable (Thai is primary market). Alternative:
```html
<link rel="alternate" hreflang="x-default" href="https://www.sandboxhotel.com/?lang=en">
```
If English should be the international default.
